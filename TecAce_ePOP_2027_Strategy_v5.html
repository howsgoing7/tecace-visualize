01 — Situation Assessment

상황 인식

우리가 아는 것
삼성은 27년형 ePOP부터 USB 수동 설치를 네트워크 업데이트로 전환하려 한다. 동시에 "New ePOP"라는 이름으로 법인이 직접 Feature와 번역 문구를 적용하는 CMS 기반 시스템을 연구 중이며, 이 프로젝트는 제일기획이 테스트하고 있다.
핵심 인텔
문제 발생 제일기획 테스트 순조롭지 않음 — TV 인터널 메모리 한계 + 전체 화면 해상도 문제
비딩 8~9월 예정 (해외법인 대상 4월 런칭 타겟)
마지막 27년형 ePOP = 기존 embedding 방식의 마지막 버전 가능성
유사 1월 TecAce 제안 내용과 현재 테스트 내용 — 큰 틀 유사, 접근 방식만 다름
이것이 의미하는 바
제일기획이 구조적 기술 한계에 부딪힌 상태에서 비딩이 5개월 앞으로 다가왔다. 이 창(window)은 TecAce가 4년간 쌓은 FW 역량 + AI 기술 차별화로 "New ePOP의 기술 파트너"를 선점할 수 있는 결정적 타이밍이다.
02 — Competitor Analysis

제일기획의 구조적 약점 분석

문제의 근본 원인
제일기획의 접근 = "완성된 4K 콘텐츠(영상/이미지)를 CMS에서 TV로 전송" → 이 방식은 구조적으로 해결 불가능한 3가지 문제를 내포한다.
메모리 과부하
고해상도 영상/이미지를 TV 인터널 스토리지에 저장 필요. TV 내장 메모리(8~16GB, 가용 2~4GB)로는 다수의 ePOP 콘텐츠 수용 불가.
해상도 깨짐
FHD/4K/8K 패널 대응 시 해상도별 리소스 각각 필요. CMS 단일 리소스 → 스케일링 과정 화질 열화 발생.
배포 속도 제약
콘텐츠 용량 大 → 네트워크 전송 시간 증가. 매장 Wi-Fi 환경에서 안정적 전송 보장 불가.
왜 에이전시로서는 풀 수 없는가
제일기획은 "콘텐츠를 만드는 회사"이지, "TV 펌웨어를 다루는 회사"가 아니다. 메모리 최적화, 네이티브 렌더링, TV 하드웨어 자원 관리는 펌웨어 레벨의 역량이 필요하다. 이것이 TecAce만의 영역이다.
03 — Asymmetric Strategy

TecAce의 비대칭 전략 — 3개 Layer

1
Hybrid AI Rendering
하이브리드 AI 렌더링
영상 대신 데이터를 전송

서버에서 초경량 데이터(JSON 구조체 + 텍스트 + 에셋 참조)만 전송하고, TV 내부에서 이를 해석하여 화면을 실시간 렌더링. TV에 탑재된 경량 언어 모델(SLM)이 수신 데이터를 해석하여 최적 레이아웃과 문구 배치를 결정, 네이티브 UI로 렌더링한다.

Phase A
즉시 가능
JSON 기반 템플릿 렌더링 엔진. LLM 없이도 현재 FW 역량으로 구현. 이것만으로도 메모리/해상도 문제 해결.
Phase B
3~6개월
SLM 기반 지능형 렌더링. 경량 모델이 레이아웃 자동 최적화, 문구 자동 현지화, 컨텍스트 기반 UI 배치 수행.
Phase C
6~12개월
완전 자율 렌더링. 온디바이스 LLM이 매장 환경, 시간대, 고객 행동 패턴까지 반영한 동적 콘텐츠 생성.
항목 제일기획 (콘텐츠 전송) TecAce (데이터 전송 + 렌더링)
전송 데이터4K 이미지/영상 (수십 MB)JSON + 텍스트 (~2KB)
TV 메모리 점유높음 (콘텐츠 수 비례 증가)최소 (렌더링 엔진만 상주)
해상도 대응패널별 리소스 필요네이티브 렌더링으로 자동 대응
매장 Wi-Fi 의존도높음 (대용량 전송)극히 낮음 (초경량 데이터)
오프라인 동작불안정엔진 탑재 후 독립 동작 가능
2
Semantic ePOP Assembly
세일즈 지식 그래프 기반 동적 조립
온톨로지 기반 추론

제품 스펙, 타겟 고객 프로필, 매장 위치, 프로모션 조건, 시간대 등의 관계를 지식 그래프로 구축. 이 관계망을 기반으로 시스템이 스스로 추론하여 "지금, 이 매장에서, 이 고객에게, 가장 효과적인 화면"을 자동 조립.

구체적 시나리오
입력: 토요일 오후 3시 + 코스트코 시애틀점 + Neo QLED 85인치 진열대
온톨로지 추론: 주말 오후 → 가족 단위 방문 높음 + 코스트코 → 번들 딜 선호 + Neo QLED 85 → 프리미엄 세그먼트
출력: 가족 시청 시나리오 배너 + 사운드바 번들 프로모션 + 24개월 무이자 QR코드
단계내용시점
1단계 — 룰 기반 매칭사전 정의된 조건-결과 매핑 테이블. "매장 유형 × 제품 카테고리 × 프로모션 유형" 조합 세팅즉시 가능
2단계 — 지식 그래프 기반 추론제품-고객-매장 관계를 그래프 DB로 구축. 새로운 조건 조합에도 추론 기반 콘텐츠 조립6개월
3단계 — 학습 기반 최적화배포 후 성과 데이터(QR 스캔율, 체류 시간)를 피드백하여 조립 로직 자동 개선12개월

03+ — Strategy Evolution

치열한 검증이 만든 무결점 3.0 아키텍처

3번의 비판, 3번의 진화
단순한 아이디어가 아닙니다. 실전에서 통할 수 있도록 치명적인 반론을 직접 만들고 해결하며 탄생한 무결점 전략입니다. 각 Layer는 초기 아이디어 → 치명적 비판 → 최종 솔루션의 사이클을 거쳐 완성됐습니다.
Layer 1 — 네트워크 & 메모리 한계 돌파
IDEA
초기 아이디어
무거운 영상 대신 2KB의 가벼운 JSON 데이터만 보내서 TV가 직접 화면을 그리게 하자. 메모리·해상도 문제 원천 해결.
→
ATTACK
치명적 비판
제일기획의 고화질 배경 영상은 결국 다운받아야 하잖아? 영업시간에 수십 대의 TV가 동시에 영상을 다운받으면 매장 네트워크가 마비된다!
→
SOLVED
최종 솔루션 — Smart Night-time Caching
영업시간엔 네트워크 점유율 0%. 무거운 영상은 새벽 2시 자동 백그라운드 캐싱. 낮엔 2KB 텍스트만 실시간 전송 → 네트워크 부하 없이 100% 화질 보장.
Layer 2 — 개인정보 침해 없는 초정밀 타겟팅
IDEA
초기 아이디어
카메라나 시간대 데이터로 고객의 연령·성별을 추측해서 맞춤형 광고를 띄우자.
→
ATTACK
치명적 비판
카메라 인식은 심각한 개인정보 침해(Privacy Violation)! 시간대만으로 고객을 추측하는 건 부정확한 헛발질이다!
→
SOLVED
최종 솔루션 — Fact-Based Proxy Targeting
개인정보 대신 '절대 틀리지 않는 팩트'에 반응. 날씨, 매장 소음 밀도, 실시간 재고 데이터("현재 3대 남음!")로 AI가 즉각 문구 조립 → 리스크 0%, 전환율 극대화.
Layer 3 — 유통사와의 채널 갈등 해결
IDEA
초기 아이디어
TV 화면에 QR코드를 띄워서 고객이 스마트폰으로 삼성닷컴(D2C)에 접속해 결제하게 만들자.

ATTACK

→
SOLVED
최종 솔루션 — Retailer-Aligned Routing
QR 스캔 시 삼성닷컴이 아닌 해당 유통사(베스트바이) 장바구니로 직결 또는 계산대 전용 바코드 발급. 삼성은 리테일 데이터, 유통사는 오프라인 매출 → 완벽한 Win-Win.
04 — Competitive Positioning

제일기획과의 포지셔닝 — The Maestro 아키텍처

핵심 메시지
"ePOP의 목표는 '감동(호객)'과 '판매(구매 전환)'를 동시에 달성하는 것입니다. 제일기획의 영상이 고객의 발걸음을 멈추게 하는 무대라면, TecAce의 다이내믹 UI는 실제 지갑을 열게 만드는 배우입니다. 이 둘은 경쟁 관계가 아니라 — AI의 지휘 아래 완벽히 공생하는 삼성 역사상 최고의 마케팅 듀오입니다."
1+1=3을 만드는 레이어 분리 아키텍처
무대(배경)와 배우(UI)의 역할을 완벽하게 분리하여, 서로의 장점을 100% 끌어올립니다. 그 위에 AI 지휘자가 타이밍을 조율합니다.

07 — Risk Management

리스크와 대응

1
제일기획이 비딩 전까지 메모리/해상도 문제를 해결할 경우
기술 경쟁에서 일부 격차가 줄어들 수 있다.
대응: 해결하더라도 "콘텐츠 전송" 방식은 근본적으로 온톨로지 기반 동적 조립이나 O2O 전환을 구현할 수 없다. 기능 범위에서 여전히 TecAce 우위.
2
삼성이 제일기획에게 단독 발주할 경우
단일 파트너로 선정될 가능성.
대응: Track A에서 TIZEN팀이 기술적 한계를 인식하도록 사전 작업. "기술 파트너"로서 제일기획과 병행 투입되는 구조를 제안.
3
온디바이스 LLM/온톨로지가 비딩 시점까지 PoC 수준에 미달할 경우
AI 기능 완성도가 시연 수준에 미치지 못할 수 있다.
대응: Phase A(JSON 렌더링)만으로도 충분히 경쟁력 있음. AI는 로드맵으로만 제시하고, 즉시 시연 가능한 것(네트워크 배포 + 템플릿 렌더링 + 다이내믹 QR)에 집중.
