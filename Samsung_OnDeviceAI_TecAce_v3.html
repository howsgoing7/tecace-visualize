Pain Point 01 · Network

건설, 물류 지하, 격오지 등 무선 통신이 단절된 환경에서는 기존 클라우드 기반 AI 문서 검색 서비스 사용이 불가능.
OnDevice AI 솔루션

매뉴얼 산재 및 검색 어려움
방대한 장비 매뉴얼과 절차서가 파편화되어 있어, 긴급 유지보수나 장애 대응 시 필요한 정보를 찾는 데 장시간 소요.

보안 아키텍처 — 4계층 완벽 보안
데이터 경계·저장 보호·전송 보호·접근 제어 — 엔터프라이즈급 보안 체계
1. 데이터 경계 (Data Boundary)
오프라인 모드 기반의 완벽한 격리
  • 질의·검색 문서·AI 응답이 디바이스 외부로 전송되지 않음
  • Air-gapped 환경 수준의 완벽한 데이터 프라이버시 보장
2. 저장 보호 (Data at Rest)
로컬 데이터 암호화 및 하드웨어 보안
  • Android Keystore 시스템을 활용한 안전한 키 관리 및 스토리지 암호화
  • 하드웨어 기반 보안 영역(TEE/SE) 연동으로 로컬 Vector DB·캐시 보호
3. 전송 보호 (Data in Transit)
OTA 업데이트 및 동기화 보안
  • 모든 관리 트래픽 및 OTA 배포 시 최신 TLS 적용 구간 암호화
  • 서버 서명된 모델·문서 패키지 검증을 통한 무결성 보장 (위변조 방지)
4. 접근 제어 (Access Control)
인증 및 권한 기반 통제
  • 관리자(사용자)와 디바이스 인증 토큰을 엄격히 이원화 (JWT 기반)
  • RBAC 적용 및 상세한 감사 추적(Audit Trail) 로깅
HIPAA · 방산 규정 · GDPR · 국내 개인정보보호법 완벽 대응
완전한 로컬 동작으로 데이터가 디바이스 외부로 전송되지 않아, 엄격한 규제 환경에서 추가적인 BAA 체결 및 보안 심사 비용을 획기적으로 절감.
